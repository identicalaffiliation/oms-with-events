## Order Managment System with events
Система представляет собой микросервисное приложение менеджмента заказов и уведомления о статусах заказа. Реализовано в рамках изучения event-driven архитектуры при помощи брокера сообщений Apache Kafka. 

### Сервисы
1. Order Service позволяет:
  * Создавать заказы.
  * Менять статус заказов (created -> paid -> shipped).

Особенность реализации: outbox паттерн для гарантии доставки: таблица заказов + событий в одной транзакции; dispatcher, который работает в фоне и раз в тик собирает батчами ивенты из таблицы и кладет их в кафку параллельно по партициям. Для предотвращения конкурентного доступа воркеров (tx для select + update) к Postgres реализован механизм FOR UPDATE SKIPPED LOCK (rows lock в PostgreSQL).

2. Notification Service позволяет:
  * Логировать(уведомлять) событие из кафки в поток STDOUT.
  * Логировать событие из кафки в таблицу логов.

Особенность реализации: proccesser, который слушает кафку и разбирает по партициям события, кладя в таблицу логов и выводя в консоль. Proccesser для предотвращения потери событий для делает fetch сообщения -> handle ивента -> commit сообщения.

Также в рамках каждого сервиса реализована корректная работа с контекстами при помощи select. Если контекст завершается, то воркеры корректно останавливают свою работу. 

### Запросы
```json
POST http://localhost:8080/orders
{
  "userId": "uuid",
  "status": "string",
  "amount": "int"
}

GET http://localhost:8080/orders/my/{userId}

PATCH http://localhost:8080/orders
{
  "orderId": "uuid",
  "status": "string"
}
```

### Конфигурация Kafka
Создан init python скрипт, который создает orders топик и partition=3 для Kafka.

Конфигурация:
```yaml
kafka:
  brokers:
    - "kafka:9092"
  topic: orders 
```

### Контейнеризация
Сервисы изолированы при помощи Docker, инфраструктура поднимается при помощи docker compose, что позволяет гибко расширять систему.

Чтобы работать с запуском и остановкой приложения были описаны таргеты в Makefile, которые позволяют не писать каждый раз сложные compose команды:
```zsh
make up # поднять приложение
make down # остановить приложение
```
